## Corner radius

Inner = outer − padding. Card `rounded={20} p={12}` → children `rounded={8}`. Cards 16–24, buttons 8–12, chips 4–8, pill = height/2.

Use radius as a design-system decision, not decoration:

- Sharp/low radius 0-6: editorial, enterprise, dense dashboard, technical UI.
- Medium radius 8-14: product UI, cards, buttons, balanced modern layouts.
- Large radius 16-24: friendly community, soft storytelling, mobile cards.
- Pill radius height/2: chips, toggles, compact CTAs, nav pills only.

Keep radius families consistent inside one output. Premium minimal usually uses subtle radius and light shadow; friendly/playful can use larger radius; UI/dashboard-like needs disciplined, repeatable radius. Do not make every object a pill.
